# 税务类目智能映射与推荐系统 (Smart Tax Category Recommendation System)

## 项目背景与痛点
在跨境数字零售业务中，不同国家和地区的税法政策差异巨大。计税引擎在计算商品税费时，需要将各类业务类目精确匹配到目标国家/地区的税务类目，并查询对应的税率。

此前系统存在以下**核心痛点**：
1. **人工匹配精度不足，容错率低**：类目繁杂，单纯依赖人工进行税务类目匹配，极易受到个人主观判断影响，准确率缺乏保障，导致税务合规风险。
2. **新增类目映射滞后，退化严重**：随着业务快速迭代，上线后大量新增类目未被及时打标映射。
3. **兜底税率滥用，削弱竞争力**：当映射缺失或失败时，系统只能使用保守的“兜底税率（通常为最高税率）”进行计税，这直接抬高了商品的前端售价，严重影响了平台的商品价格竞争力。

---

## 解决方案
为了解决上述痛点，本项目构建了一套**基于大模型（LLM）与知识库（RAG）结合的多 Agent 协同系统**，实现了从离线历史数据清洗到新增业务类目自动映射的完整闭环。

### 核心逻辑流 (多 Agent 协同)
系统整体设计为多步骤的 Agent 流水线，核心包含以下组件：
1. **RAG 检索模块 (RAG Engine)**：将全球各地区海量税法规则、类目定义以及历史优质的映射记录向量化，存入知识库。
2. **推荐决策 Agent (Mapper Agent)**：负责长链推理计算。给定一个业务类目的描述及属性，调用 RAG 模块获取候选税务类目和相似历史判断上下文，结合 Prompt 深度推理各类目间的关联与排除逻辑，输出 Top-N 推荐映射并附带推荐理由与置信度。
3. **校验核查 Agent (Reviewer/Critic Agent)**：扮演“交叉检查者”角色。对 Mapper Agent 给出的高优结果再一次结合“排他性税法规避条件”进行反思和二次校验，避免大模型幻觉，并对历史人工映射做回溯审计。
4. **分布式调度链路核心**：设计定时推荐全链路系统，通过分布式调度（如 XXL-JOB/Airflow）剥离离线推理算力，通过并行处理支撑海量数据的全量洗刷，最后同步落库至业务系统。

---

## 优化过程 (Tuning & Optimization)
- **幻觉治理与 Prompt 调优**：在初期阶段，发现大语言模型易出现“泛化灾难（幻觉）”，导致相近税务类目混淆。我们通过**结果差异分析**，定位了易混淆的边界情况。
- **构建正负样本库 (Few-Shot 增强)**：针对易错类目，定向爬取和人工筛选出标准的“正样本（明确属于）”和“负样本（极易混淆但不属于）”。将其沉淀至 Few-Shot 样本库并在 Prompt 动态加载，大幅减少了边缘场景的大模型幻觉。
- **一致性与稳定性提升**：通过 Temperature 调控及引入 Reviewer Agent 的 Double-Check 机制，使推荐的准确性和输出结构的稳定性达到了工程落地标准。

---

## 最终结果与业务收益
1. **历史数据清洗**：系统离线校验了 **3.77 万条** 历史人工匹配记录，精准识别出约 **6% 的错配率**，并自动输出了结构化的错因分析报告，极大提升了历史数据的资产质量。
2. **新增类目映射**：面向未覆盖的 **3.1 万条** 新增业务类目提供智能推荐，闭环上线后，**兜底税率的使用率暴降 87%**。
3. **商业价值转化**：兜底税率占比的大幅下降，使得海量商品的最终计税成本回归合理区间，直接有效增强了平台商品的前端价格竞争力，带动了订单转化率的提升。
